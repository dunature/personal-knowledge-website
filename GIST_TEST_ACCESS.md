# 访问 Gist 集成测试页面

## 🚀 快速访问

### 方法 1: 通过导航菜单
1. 启动开发服务器：
   ```bash
   npm run dev
   ```

2. 打开浏览器访问：
   ```
   http://localhost:5173
   ```

3. 点击顶部导航栏的 **"Gist集成测试"** 链接

### 方法 2: 直接访问
```
http://localhost:5173/gist-integration-test
```

---

## 📋 测试页面功能

测试页面包含以下功能：

### 1. 状态指示器
- **模式状态**: 显示当前是拥有者模式还是访客模式
- **同步状态**: 显示同步状态和最后同步时间

### 2. 快速操作
- **打开配置向导**: 配置 GitHub Token 和模式
- **打开迁移向导**: 迁移本地数据到 Gist
- **清除所有数据**: 重置所有配置和数据
- **清除测试结果**: 清除测试结果显示

### 3. 功能测试按钮
1. 测试加密工具
2. 测试 AuthService
3. 测试 CacheService
4. 测试 PermissionService
5. 测试 MigrationService
6. 测试 SyncService

### 4. 测试结果显示
- 每个测试的成功/失败状态
- 详细的返回数据（JSON 格式）
- 错误信息（如果测试失败）

## 🔍 如果页面仍然空白

### 检查步骤：

1. **检查浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签是否有错误信息
   - 查看 Network 标签检查资源加载

2. **检查开发服务器**
   - 确认服务器正在运行
   - 查看终端是否有编译错误

3. **清除缓存**
   ```bash
   # 停止服务器 (Ctrl+C)
   # 清除缓存
   rm -rf node_modules/.vite
   # 重新启动
   npm run dev
   ```

4. **检查路由**
   - 确认 URL 是 `/gist-integration-test`
   - 尝试访问其他页面（如 `/components`）确认路由工作正常

## 🐛 常见问题

### 问题 1: 404 Not Found
**解决方法**: 确认 URL 拼写正确，应该是 `/gist-integration-test`

### 问题 2: 白屏但无错误
**解决方法**: 
1. 检查是否有 React 错误边界捕获了错误
2. 尝试访问 `/` 主页确认应用正常运行
3. 重新启动开发服务器

### 问题 3: 组件加载失败
**解决方法**:
1. 检查所有依赖的组件是否存在
2. 运行 `npm install` 确保依赖安装完整
3. 检查 TypeScript 编译错误
