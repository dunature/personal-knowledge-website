## 🚀 快速开始

### 第一次使用

1. **启动应用**
   ```bash
   npm install
   npm run dev
   ```
   访问 http://localhost:5173

2. **添加第一个资源**
   - 点击右上角 "all time" 按钮
   - 点击 "+ 添加资源"
   - 填写信息并保存

3. **创建第一个问题**
   - 滚动到问答板区域
   - 点击 "查看全部问题"
   - 点击 "+ 添加大问题"
   - 填写信息并保存

## 📚 资源管理

### 添加资源

#### 步骤
1. 点击 **"all time"** 展开资源区域
2. 点击 **"+ 添加资源"**
3. 填写表单：
   - 标题 *（必填）
   - 链接 *（必填）
   - 封面图片URL（可选）
   - 分类
   - 作者
   - 推荐语
4. 点击 **"保存"**

#### 封面图片来源

**YouTube视频缩略图**
```
https://img.youtube.com/vi/视频ID/maxresdefault.jpg
```

**占位图（临时使用）**
```
https://via.placeholder.com/320x180/0047AB/FFFFFF?text=标题
```

**免费图床**
- Imgur: https://imgur.com
- ImgBB: https://imgbb.com

> 详细说明：[RESOURCE_COVER_GUIDE.md](./RESOURCE_COVER_GUIDE.md)

### 编辑资源

1. 鼠标悬停在资源卡片上
2. 点击右上角 **三点菜单**
3. 选择 **"编辑"**
4. 修改后保存

### 删除资源

1. 点击三点菜单
2. 选择 **"删除"**
3. 确认删除

### 筛选资源

- **按分类**：点击分类标签
- **按标签**：点击资源卡片上的标签
- **搜索**：输入关键词搜索
- **排序**：选择排序方式

## 💬 问答管理

### 大问题

#### 添加大问题
1. 点击 **"查看全部问题"**
2. 点击 **"+ 添加大问题"**
3. 填写：
   - 问题标题 *
   - 问题描述（支持Markdown）
   - 分类
   - 状态
4. 保存

#### 编辑大问题
1. 点击问题打开详情
2. 点击顶部 **"编辑"** 按钮
3. 修改后保存

#### 删除大问题
1. 在详情页点击 **"删除"** 按钮
2. 确认（会删除所有小问题和回答）

### 小问题

#### 添加小问题
1. 打开大问题详情
2. 点击 **"+ 添加小问题"**
3. 填写标题和状态
4. 保存

#### 编辑小问题
1. 点击小问题的 **"编辑"** 按钮
2. 修改后保存

#### 删除小问题
1. 点击 **"删除"** 按钮
2. 确认（会删除所有回答）

### 回答

#### 添加回答
1. 展开小问题
2. 点击 **"添加回答"**
3. 使用Markdown编辑器输入
4. 保存

#### 编辑回答
1. 点击回答的 **"编辑"** 按钮
2. 修改后保存

#### 删除回答
1. 点击 **"删除"** 按钮
2. 确认

### THE END 总结

1. 在问题详情中找到 "THE END - 最终总结"
2. 点击 **"编辑"**
3. 输入总结（支持Markdown）
4. 保存

## ❓ 常见问题

### 数据相关

**Q: 数据会保存吗？**  
A: 当前版本数据在内存中，刷新会重置。未来版本会支持持久化。

**Q: 如何备份数据？**  
A: 当前版本暂不支持导出。建议定期截图或复制重要内容。

### 图片相关

**Q: 图片不显示？**  
A: 检查URL是否正确，是否可公开访问。建议使用图床服务。

**Q: 可以上传本地图片吗？**  
A: 当前需要先上传到图床。未来版本会支持本地上传。

**Q: 推荐的图片尺寸？**  
A: 320x180像素（16:9比例）

### 功能相关

**Q: 如何批量操作？**  
A: 当前需要逐个操作。批量功能在开发计划中。

**Q: 支持多人协作吗？**  
A: 当前是单用户系统。

**Q: 可以导出为PDF吗？**  
A: 当前不支持。可以使用浏览器打印功能。

### 技术相关

**Q: 启动失败？**  
A: 确保Node.js >= 18，已运行npm install

**Q: 端口被占用？**  
A: 修改vite.config.ts中的端口号

**Q: 构建失败？**  
A: 运行npm run build查看错误信息

## 💡 使用技巧

### 资源管理技巧

1. **使用分类**：为资源设置合适的分类，便于筛选
2. **添加标签**：使用内容标签标记资源特点
3. **写推荐语**：记录为什么保存这个资源
4. **定期整理**：删除过时或不再需要的资源

### 问答管理技巧

1. **问题分解**：将大问题分解为多个小问题
2. **及时记录**：遇到问题立即记录，避免遗忘
3. **时间线回答**：按时间顺序记录解决过程
4. **写总结**：问题解决后写THE END总结

### Markdown技巧

1. **使用标题**：组织内容结构
2. **代码高亮**：指定语言获得语法高亮
3. **插入图片**：使用图床链接
4. **善用列表**：清晰展示要点
